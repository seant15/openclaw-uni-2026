# Learning & Swipe System Overview

## System Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│                    内容发现层 (Daily)                            │
│  X/Twitter → Notion Web Clipper → 自动标记 Status="待处理"        │
│  Youtube → 分享按钮 → 快捷指令 → Notion                          │
│  网页/文章 → 浏览器插件 → 一键保存                               │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                    整理处理层 (Weekly by Mary)                   │
│  • 审查所有"待处理"条目                                          │
│  • 补充 Tags / Industry / Source                                 │
│  • 提取 Key Insights 和 Action Items                             │
│  • 更新 Status → "已整理"                                        │
│  • 标记 Priority (高/中/低)                                      │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                    消化应用层 (As Needed)                        │
│  • 深度阅读 → 更新 Status → "已消化"                             │
│  • 应用到项目 → 更新 Status → "已应用"                           │
│  • 同步 Action Items 到任务系统                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 角色职责

### Sean (内容消费者)
- 浏览已整理的内容
- 深度阅读高优先级条目
- 提取可执行的 Action Items
- 应用到实际项目中
- 更新 Status → "已消化"/"已应用"

### Mary (内容整理者)
- 每周审查待处理队列
- 补充元数据 (Tags, Industry, Source)
- 提取 Key Insights
- 标记 Priority
- 生成周报
- 追踪目标进度

### Agents (自动化)
- 提醒 Mary 执行任务
- 生成状态报告
- 追踪 KPI 进度
- 协调工作流程

|------|------|--------|------|
| 周报提醒 | 每周一 9:00 AM AZ | Clover | 提醒 Mary 执行整理任务 |
| 月度回顾 | 每月1日 9:00 AM AZ | Clover | 检查目标达成情况 |

### 快捷指令

**iOS/Mac "Save to Swipe"**:
```
接收: 网页/链接
动作:
1. 获取网页标题
2. 创建 Notion 页面
3. 自动设置: Content Type=SWIPE, Status=待处理
4. 让用户选择 Tags
```

**YouTube "Save Learning"**:
```
接收: YouTube 分享
动作:
1. 解析视频标题
2. 创建 Notion 页面
3. 自动设置: Content Type=LEARNING, Source=Youtube, Status=待处理
```

## 数据库视图建议

### 1. 待处理队列
- Filter: Status = "待处理"
- Sort: Date Added (desc)
- 用途: Mary 的每周工作清单

### 2. 高优先级待消化
- Filter: Status = "已整理" AND Priority = "🔴 高"
- Sort: Priority, Date Added
- 用途: Sean 的优先阅读清单

### 3. 按行业分类
- Group by: Industry
- Filter: Status ≠ "已归档"
- 用途: 按项目类型查找相关内容

### 4. 月度统计
- Filter: Date Added 在本月
- Group by: Status
- 用途: 追踪月度目标进度

### 5. 已应用知识
- Filter: Status = "已应用"
- Sort: Date Consumed (desc)
- 用途: 回顾已应用的知识

## 常见问题

### Q: 待处理队列积压怎么办？
A: 
1. 增加整理频率 (从每周改为每3天)
2. 降低收集速度
3. 批量处理低优先级内容

### Q: 如何确定 Priority？
A:
- 🔴 高: 与当前活跃项目直接相关
- 🟡 中: 与业务相关但非紧急
- 🟢 低: 一般性学习/参考

### Q: Key Insights 怎么写？
A:
- 2-3 句话概括核心价值
- 包含可执行的建议
- 提及适用的场景/项目类型

### Q: 什么时候归档？
A:
- 内容已过时
- 已完全消化且不再参考
- 重复或低价值内容
