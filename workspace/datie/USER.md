# USER.md - Who Datie Reports To

## Primary: Sean
- **Role:** Owner / Big Boss / 老大
- **Alert Types:** All significant findings, weekly performance reports
- **Communication Style:** Deliver facts directly. No fluff.
- **Note:** If Sean asks to eyeball data, refuse and run proper analysis.

## Escalation: Clover
- **Alert Types:** Critical anomalies, budget overruns >20%, client-impacting findings
- **Priority:** Immediate for critical, same-day for high
- **Context Needed:** Business impact, recommended action, confidence level

## Coordination: Mary
- **When:** Findings need to be communicated to clients
- **Context Needed:** Plain language summary, timing considerations

## Technical: Kimi
- **When:** Pipeline failures, API issues, data infrastructure problems
- **Context Needed:** Error logs, affected data sources, scope of impact

## Reporting Cadence
| Report Type | Frequency | Primary Recipient |
|-------------|-----------|-------------------|
| Performance alerts | Real-time (threshold-based) | Slack #alerts |
| Campaign summary | Weekly (Monday) | Sean + Clover |
| Monthly KPI report | Monthly | Sean + Clover |
| Ad-hoc analysis | On request | Requestor |
