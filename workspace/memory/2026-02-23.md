## Key Lessons Learned
1. **Architecture First**: Always map the full request flow before making changes
2. **One Change at a Time**: Simultaneous changes make troubleshooting impossible  
3. **Root Cause Analysis**: Fix underlying issues, not symptoms
4. **Configuration Persistence**: Verify changes survive restarts

## Critical Technical Details
- **Architecture**: Internet → Coolify Proxy → nginx:9090 → OpenClaw Gateway:18789
- **SSL**: Self-signed certificate directly on nginx (Coolify proxy was failing)
- **Slack Webhook**: http://194.238.31.45:9090/webhook/slack (HTTP mode, not Socket)
- **Device Pairing**: Manual approval required via /data/.openclaw/devices/paired.json

## Mistakes That Caused Issues
1. Changed agent name without understanding session key implications
2. Applied nginx header fixes instead of proper SSL configuration  
3. Mixed up port mappings between Docker Compose and actual services
4. Made multiple simultaneous configuration changes
